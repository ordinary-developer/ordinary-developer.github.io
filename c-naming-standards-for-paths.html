C++ naming standards for paths

C++ naming standards for paths

Commont thoughts about the theme

Very often tasks such as function and variable naming for work with paths (files, directories etc.) can be arisen. Here is a question in SO.

I'll try to write shortly about these name conventions (that are convenient for me).

Let's suppose that we have the next paths:

Windows: c:\users\user\widget\foo.src

Posix: /users/user/widget/foo.src

Part Windows Posix
full_path (abosolute_path, full_filename) c:\users\user\widget\foo.src /users/user/widget/foo.src
filename foo.src foo.src
basename foo foo
fileext (filename_ext) src src
dirpath (parentpath) c:\users\user\widget /users/user/widget
dirname (parentname) widget widget
rootpath c:\ /
volumename c <empty>

Qt approach

The QFileInfoclass has some methods: absoluteFilePath(), baseName(), fileName() etc. The QDir class has the rootPath() and root() methods and the QFile class has some usefull methods.

According to this class we can make the next suggestions:

Let's suppose that we have the next paths:

Windows: c:\users\user\widget\foo.src

Posix: /users/user/widget/foo.src

Part Windows Posix
absolute_path (abosolute_path, full_filename) c:\users\user\widget\foo.src /users/user/widget/foo.src
filename foo.src foo.src
basename foo foo
? src src
absolute_dir (dir) c:\users\user\widget /users/user/widget
? widget widget
rootpath c:\ /
? c <empty>

Everything is made up for us

But may be it is better to use Boost.FileSystem (since cpp17 it is availible in std::filesystem). Boost.FileSystem (std::filesystem) has the next name conventions.

Let's suppose we have the next paths:

Windows path: c:\foo\bar\baa.txt

Posix path: /foo/bar/baa.txt

Part Windows Posix
Root name c: <empty>
Root directory \ /
Root path c:\ /
Relative path foo\bar\baa.txt foo/bar/baa.txt
Parent path c:\foo\bar /foo/bar
Filename baa.txt baa.txt
Stem baa baa
Extension .txt .txt

And we can add the next item: (added by me)

PartWindowsPosix
Fullpath c:\foo\bar\baa.txt /foo/bar/baa.txt
