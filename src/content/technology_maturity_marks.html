Technology maturity marks

Sometimes I want to understand which technology is mature and can be used in production and which one is not. I defined several maturity marks for myself.

  • Thoughtworks radar (https://www.thoughtworks.com/radar)
    If the technology ("technique", "tools", "Platforms", "language and frameworks") is in the "Adopt" section, then it is mature.
  • - JetBrains ide list (https://www.jetbrains.com/) If JetBrains has an IDE for the language, then the language is mature and can be popular on "the market".

And yet another "tricky trick".

If language (environment etc.) version you use is less than one from the current stable Debian Linux distributive, it can be said, that is is obsolete.
